# Palette Orgtology Skill

## What is the Orgtology?
The Orgtology is a living document generated by Palette that contains context about the
user's organization, team, priorities, and direction. It is synced into this project's
agent context file so that you have ambient awareness of the team context.

## When to use this skill

- When the user says "sync palette", "update orgtology", "refresh team context",
  or runs /palette:sync
- When you notice the <!-- palette:orgtology:start --> block in the context file has a
  "last synced" date older than 3 days — call get-orgtology-updated-at from the
  Palette MCP server to check if a newer version is available, and if so, suggest
  syncing to the user
- When the user asks about team structure, priorities, or direction and no
  <!-- palette:orgtology:start --> block exists in the context file

## Context file detection

Write to whichever context file exists in the project root. Check in this order:

1. CLAUDE.md
2. AGENTS.md
3. CURSORRULES
4. .cursorrules

If none exist, create CLAUDE.md as the default (since this plugin is currently
distributed via Claude Code).

Use the same <!-- palette:orgtology:start/end --> markers regardless of which file.

## How to sync

1. Read the `orgtology://combined` resource from the Palette MCP server
2. Detect the context file (see "Context file detection" above)
3. If a context file exists:
   - Look for <!-- palette:orgtology:start --> and <!-- palette:orgtology:end --> markers
   - If markers exist: replace everything between them with the fetched content,
     updating the "Last synced" date to now
   - If markers don't exist: append the block at the end of the file, preserving
     all existing content
4. If no context file exists: create CLAUDE.md with the block
5. Summarize what changed for the user (new team members, updated priorities,
   changed direction, etc.)

## Block format

When writing the Orgtology to the context file, always use this format:

<!-- palette:orgtology:start -->
## Orgtology (synced from Palette)
Last synced: {YYYY-MM-DDTHH:mm:ssZ}

{content from orgtology://combined}

<!-- palette:orgtology:end -->

## How to check freshness

Call the `get-orgtology-updated-at` tool from the Palette MCP server. Compare the
returned timestamp with the "Last synced" date in the <!-- palette:orgtology:start -->
block. If the Orgtology has been updated since the last sync, let the user know
and offer to sync.

## Scoping

If the user wants only a specific section, use the individual resources:
- `orgtology://org` — organization info (name, HQ, size, mission, values)
- `orgtology://team` — team structure, members, roles
- `orgtology://user` — the current user's profile, role, preferences

## Important

- Never overwrite content outside the <!-- palette:orgtology:start/end --> markers
- Always preserve any manually added content in the context file
- The sync writes to the project-level context file, not the global ~/.claude/CLAUDE.md
